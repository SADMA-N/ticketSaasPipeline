# US-6.1 — Real-time Socket Events

## 1. Big Picture

Before this change, when a ticket was being processed, the client had no idea what was happening — they had to keep asking "is it done yet?" (polling).

Now, the server **pushes messages** to the client automatically — "Phase 1 started", "Phase 2 done", "completed" — without the client asking. This is done using **Socket.IO**, which keeps a live connection open between server and client.

---

## 2. Step-by-Step Flow

```
Client opens browser
  → connects to Socket.IO server
  → sends "subscribe" message with task ID
  → server puts client in a "room" for that task

Worker picks up a ticket job
  → emits "started"          → client receives it instantly
  → runs Phase 1
  → emits "phase_1_started"  → client gets it
  → Phase 1 finishes         → emits "phase_1_complete"
  → runs Phase 2
  → emits "phase_2_started"
  → Phase 2 finishes         → emits "phase_2_complete" + "completed"

Client sees all these in real-time, no polling needed
```

---

## 3. Line-by-Line — `src/socket/emitter.ts` (NEW FILE)

```typescript
let io: Server;
```
`io` is the Socket.IO server instance. Stored outside the function so all files can use it.

```typescript
export function initSocket(httpServer: HttpServer) {
  io = new Server(httpServer, { cors: { origin: "*" } });
```
Attaches Socket.IO to the existing HTTP server. `cors: { origin: "*" }` = allow any client to connect.

```typescript
  io.on("connection", (socket) => {
    socket.on("subscribe", (taskId: string) => socket.join(`task:${taskId}`));
  });
```
When a client connects and sends `"subscribe"` with a task ID → put them in a **room** named `task:abc123`. Only clients in that room receive events for that task.

```typescript
export function emitSocketEvent(taskId, event, metadata = {}) {
  const payload = {
    task_id: taskId,
    event,
    timestamp: new Date().toISOString(),
    metadata: Object.keys(metadata).length ? metadata : null,
  };
  io.to(`task:${taskId}`).emit(event, payload);
}
```
Sends an event to **only** the clients subscribed to this task's room. Every event includes `task_id`, event name, timestamp, and optional metadata.

## 4. Line-by-Line — `src/app.ts` (CHANGED)

Before:
```typescript
app.listen(PORT, ...)
```

After:
```typescript
const httpServer = createServer(app);  // wrap Express in HTTP server
initSocket(httpServer);                // attach Socket.IO to same server
httpServer.listen(PORT, ...)           // start listening
```

**Why?** Socket.IO needs the raw HTTP server to attach to. `app.listen()` creates one internally and hides it — you can't pass it to Socket.IO. `createServer(app)` gives you control of it.

## 5. Line-by-Line — `processor.ts` (KEY EVENTS)

```typescript
emitSocketEvent(taskId, "started");
```
Fired right after task moves to `"processing"`. Tells client: "we picked up your ticket."

```typescript
if (task.phase1Retries > 0) {
  emitSocketEvent(taskId, "retry", { phase: "phase_1", attempt: task.phase1Retries + 1 });
}
```
`phase1Retries > 0` means this is NOT the first attempt — it's a retry. Tells client: "Phase 1 failed before, trying again. This is attempt #2 (or 3)."

```typescript
emitSocketEvent(taskId, "phase_1_started");
// ... run Phase 1 ...
emitSocketEvent(taskId, "phase_1_complete");
```
Bracket the actual AI call — client sees start and end separately.

```typescript
emitSocketEvent(taskId, "phase_2_complete");
emitSocketEvent(taskId, "completed");
```
Two events: one for "Phase 2 AI call done", one for "entire job done." Client can react to either.

## 6. Tricky Parts Explained

**What is a "room"?**
Think of it like a WhatsApp group. `socket.join("task:abc123")` = client joins the group for that task. `io.to("task:abc123").emit(...)` = send message to everyone in that group. Other tasks' clients don't receive it.

**Why `task.phase1Retries + 1` for attempt number?**
`phase1Retries` is incremented AFTER the retry check. So when checking, it still holds the OLD count.
- Before attempt 1: retries = 0 → no retry event
- Before attempt 2: retries = 1 → retry event, attempt = 1+1 = 2 ✅
- Before attempt 3: retries = 2 → retry event, attempt = 2+1 = 3 ✅
